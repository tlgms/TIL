# 예외처리

# 01.

이 방법은 내가 선배에게 배운 가장 처음의 에러 처리 방법이다. @ResponseStatus를 통해 http 메소드를 반환하고 RuntimeException을 통해 예외를 발생시키는 방법이다.

```java
@ResponseStatus(HttpStatus.NOT_FOUND) // 404
public class NotFoundException extends RuntimeException {

}
```

원하는 예외를 만들고 @ResponseStatus에 반환해야하는 메소드를 적고 RuntimeException 상속받으면 예외 처리가 가능한 클래스가 만들어진다. 

```java
throw new NotFoundException();
```

위와 같이 예외를 발생시킬 수 있다.

---

> 2번부터는 Handler를 통한 예외처리 방법이다. 
handler가 아래와 같이 정의되어 있다는 가정으로 설명을 진행한다.
```java
@ExceptionHandler(BaseException.class) // 정의한 Exception 클래스를 잡는다.
protected ResponseEntity<ErrorResponse> handlerDcsException(final BaseException e) {
	return new ResponseEntity<>(new ErrorResponse(여기의 내용은 방법마다 다름[상태코드,에러메시지]), HttpStatus.valueOf(여기의 내용은 방법마다 다름[상태코드]));
}
```

# 02.

기본이 되는 예외클래스를 아래와 같이 정의해주면 예외는 ErrorCode의 타입으로 정의될 것이다.

```java
@Getter
@RequiredArgsConstructor
public class BaseException extends RuntimeException {

	private final ErrorCode errorCode;

}
```

enum 클래스를 통해 예외를 정의하고,

> enum으로 예외를 정의하는 이유는 요구사항이 바뀌었을 때 바꾸기 쉬우며, 
예외를 고정시킬 수 있다. (통일성 유지)

```java
@Getter
@AllArgsConstructor
public enum ErrorCode {

	POST_NOT_FOUND(404,"게시글을 찾을 수 없습니다.");
	
	private final int status;
	private final String message;

}
```

enum 클래스의 필드를 가져와서 정의하는 방식으로 예외를 선언한다.

```java
public class NotFoundException extends BaseException {

	public final static NotFoundException EXCEPTION = new NotFoundException();

	public NotFoundException() {
		super(ErrorCode.POST_NOT_FOUND);
	}

}
```

EXCEPTION 변수를 통해서 생성자를 빠르게 불러올 수 있기 때문에 아래와 같이 예외를 발생시킬 수 있다.

```java
throw NotFoundException.EXCEPTION;
```

> EXCEPTION 필드를 정의하지 않고 예외를 발생시키는 것도 가능하다.
```java
throw new NotFoundException();
```

# 03.

아까와는 다르게 enum으로 정의하는 것이 아닌 바로 필드로 정의하는 방법을 사용한다.

```java
@Getter
@RequiredArgsConstructor
public class BaseException extends RuntimeException {
	
	private final int status;
	private final String message;

}
```

이번의 방법은 예외를 발생시킬 클래스에서 상태코드를 지정해주는 방식이다. 
메시지는 발생할때 상황에 맞게 적을 수 있다. 

401 에러가 발생하는 모든 상황에서 사용할 수 있다는 장점이 있다. 하지만 요구사항이 바뀌어 메시지를 수정해야한다면 많은 코드를 모두 직접 수정해야한다는 단점이 존재한다.

```java
public class NotFoundException extends BaseException {

	public NotFoundException(String message) {
		super(404, message);
	}

}
```

예외는 아래와 같이 발생시킬 수 있다.

```java
throw new NotFoundException("게시글을 찾을 수 없습니다.");
```

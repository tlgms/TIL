## CORS 정책의 한계와 백엔드 보안
1. **CORS는 브라우저 전용 보안 메커니즘**
• 적용 대상: 웹 브라우저(SOP, CORS 정책 적용)
• 목적: 사용자를 보호하기 위해 브라우저가 자바스크립트 등을 통한 교차 출처 요청을 통제함.
2. **curl / Postman이 CORS를 우회하는 이유**
• 동작 방식: 브라우저 환경을 거치지 않고 직접 서버로 HTTP 요청을 전송함.
• 결과: 브라우저의 보안 검사(Preflight Request 등)가 작동하지 않음.
• 서버의 인식: 서버는 이 요청을 일반적인 서버 대 서버(Server-to-Server)' 통신으로 인식하여 CORS로 차단할 수 없음.
3. **결론 및 대책**
• CORS는 만능 보안책이 아니며, 브라우저만을 위한 방어벽임.
• 따라서 백엔드 서버는 추가적인 인증(Authentication)과 인가(Authorization) 기능을 반드시 별도로 구현해야 함.
